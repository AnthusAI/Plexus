# Documentation Surface Audit

This audit inventories every markdown, agent-instruction, and rule surface in
the repository and classifies it against the cleanup plan in
`/Users/ryan/.cursor/plans/docs_kb_cleanup_b85ebd28.plan.md`. It is the
input to the migration, thinning, and cleanup tasks under epic
`plx-db0bd9` (Documentation knowledge base cleanup and DRY agent
instructions).

## Classifications

| Code | Meaning |
|------|---------|
| `KB`        | Canonical agent knowledge base content. Belongs in `documentation/agent/<namespace>/` with YAML frontmatter. |
| `POINTER`   | Thin entry point that should redirect to the canonical KB instead of duplicating it. |
| `SUPPL`    | Human-facing supplemental reference (Sphinx, diagrams, design notes). Not first-class agent context. |
| `GEN`       | Generated artifact (logs, planning dumps). Should be cleaned and ignored. |
| `OBSOLETE`  | Stale content describing systems that no longer exist; remove. |
| `RULES`     | Repository governance rules that stay where editors and agents already look for them, but should be DRY. |
| `KEEP`      | Already in the right place and shape; no action. |

## Frontmatter standard

All files under `documentation/agent/` MUST start with YAML frontmatter:

```yaml
---
id: mcp.execute-tactus.overview          # canonical key consumed by plexus.docs.get
title: execute_tactus Overview
summary: How agents interact with Plexus through the single execute_tactus MCP tool.
namespace: mcp
status: canonical                         # canonical | draft | deprecated
disclosure: overview                      # overview | reference | cookbook | deep-dive
audience: agent
tags: [mcp, tactus, runtime]
related:
  - mcp.discovery
  - mcp.read-apis
---
```

`_index.md` files use the same frontmatter and act as namespace landing
pages for both humans and agents (returned by `plexus.docs.list` filtered
to namespace).
